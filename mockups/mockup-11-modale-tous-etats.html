Markup & règles CSS du refactor

Tous les écrans suivent ce squelette uniforme :

  • .conseiller-card : display: grid; grid-template-rows: auto 1fr auto; + height: 620px; max-height: calc(100dvh - 64px);
  • .modal__screen : display: contents → les enfants __head / __body / __foot deviennent les vrais grid items, pas le wrapper.
  • .modal__head : grid-row: 1 · badge seul. Toujours en haut, à la même hauteur sur tous les écrans.
  • .modal__body : grid-row: 2 · absorbe l'espace restant, overflow-y: auto pour scroller si déborde. Contient un wrapper .modal__body-content (flex: 1; justify-content: center) qui centre le contenu, et optionnellement une .progress en dernier enfant.
  • .progress : position: sticky; bottom: 0 · présente uniquement sur s1, collée en bas du body, reste visible même si le contenu scrolle.
  • .modal__foot : grid-row: 3 · footer (CTA, lien, hint). Toujours en bas.
  • .modal__close : position: absolute top-right, en dehors du grid. Visible sur tous les écrans.

Plus de position: absolute sur le badge, plus de margin-top: auto sur des éléments arbitraires, plus de wrappers ad-hoc par écran. La règle globale .conseiller-card__inner { position: relative } qui polluait les fixes peut rester — on ne s'en sert plus comme conteneur principal.
